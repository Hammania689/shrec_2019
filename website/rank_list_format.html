Rank list format for SHREC 2019 - Extended 2D Scene Image-Based 3D Scene Retrieval

The participants should submit one separate rank list file for each 2D scene image in the query set. The file must be an ASCII file; the filename should be the same as the query model and does not contain any file extension.

Each line of the file should contain a 2D image-3D scene model distance in the order of increasing dissimilarity and the index number of a retrieved target model. The distance and the name should be separated by white space.

Example rank list format for the query image 1.png:
The filename of the rank list should be 1 with no file extension.
The content of the rank list should be as follows:

0.006828792688283 107

0.010065110202301 815

0.010570069229078 102
0.010597355637337 155
.......
.......

The files should be collected in a single archive file and the name of the archive file should indicate the retrieval method. The archive file should be sent via e-mail Hameed Abdul. If the participants have more than one method, they should submit an archive file for each method (the filename should indicate the method). Example archive file for the method X should have a name method_X.zip or method_X.rar. The file should contain either 21,000 (test) or 30,000 (all) rank lists for all the query scene images.
